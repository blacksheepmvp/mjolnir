# Mjolnir 0.2.8 Profile Spec

## Goals
- Make Mjolnir simple for first-time dual-screen owners while preserving power-user control.
- Introduce profiles bound to the current default home.
- Provide a visual, quick profile switcher via notification and quick tiles.
- Add a robust debug screen and error capture.

## Core Concepts
- **Profile**: A named configuration tied to a default home package. Fields:
  - `name`
  - `description`
  - `defaultHomePackage`
  - `topApp`
  - `bottomApp`
  - `gesturePreset`
  - `isActive`
  - `isDisabled`
  - `isLocked` (read-only)
  - `lastUsedTimestamp`
- **Per-home state**: Each default home has its own profile list and last-used stack.
- **Inactive state**: If no profile exists for current default home, Mjolnir is inactive.

## Storage Layout
- Base: `/android/data/xyz.blacksheep.mjolnir/`
- Profiles folder: `/profiles/{home.package}/`
  - `profile-name.cfg` (human-editable key=value, with comments)
- Profile folders are created/refreshed when app icons are cached.
- Removing a home app **must not** delete its folder or profiles.

## Profile Rules
- Profiles are **bound to the active default home**.
- Switching default home:
  - If profiles exist: activate last-used profile.
  - If none: Mjolnir becomes inactive.
- If a profile references missing packages:
  - Prompt user to select replacements.
  - If skipped: mark profile `disabled`.
  - If Play Store link exists: open store page.
- `disabled` profiles remain listed but **do not** enter last-used stack.
- Users can **deactivate** Mjolnir per default home (turn off without deleting profiles).

## Migration (0.2.7 -> 0.2.8)
- Convert current config into a profile, set active.
- Import current gesture set as a profile preset reference ("custom" or migrated name).

## Onboarding (New, Simplified)
1. **Welcome**: what Mjolnir is + permissions.
2. **Profile Manager** for current default home.
3. If no profile exists, prompt to create one.
4. If Mjolnir is default home with no profile, open dual-screen setup mode.
5. A “Change Default Home” button is always available from the manager.

## 0.2.8 Behavior Change: No Basic/Advanced Split
- Remove the Basic/Advanced mode split.
- If Mjolnir is set as default home, provide a **No‑Gestures** mode:
  - No permissions, no interception, single Home behavior only.
  - User selects the single behavior:
    - Default: **BOTH: Auto**
    - Options: **FOCUS: Auto**, **FOCUS: Top App**

## Profile Manager UI
- Card-style list (same visual system as gesture picker).
- Options: Select, Edit, Copy, Delete, Lock/Unlock.
- Context menu via long-press or X button.
- Profiles for other default homes are viewable/editable.

## Notification + Quick Tiles (Spec)
- **Tap notification body**: toggle Mjolnir on/off for current default home.
- **Action buttons**:
  - Profiles (opens carousel UI)
  - Default Home (opens home picker)
  - Settings (opens Mjolnir settings)
- **Prev/Next actions** in notification to cycle active profile.
- **Notification card**: a visual card showing the active profile.
- **Quick tiles**:
  - Profile Switcher
  - Default Home Changer
- **Rename tile**: “Safetynet/Debug” -> **“Mjolnir / Safe Mode”**.

## Debugging / Diagnostics
- New debug screen (or tab inside safetynet):
  - Active profile
  - Active gesture preset
  - Permissions status
  - Helper services status
  - Notification status + contents
  - Error log window (last ~25 lines)
- Error capture: log errors into an in-app buffer for this view.
- Safetynet hosts tabs: `SafetyNet` / `Debug`.
- Embedded launcher:
  - App dropdown + custom intent
  - Buttons: Launch to Top / Launch to Bottom
- Debug/log window rendered on 2nd display when present.
- Use high-legibility monospace font.

## Profile Switcher Carousel
- Carousel UI with large visual cards.
- Switching profile does **not** auto-launch top/bottom apps.
- Profiles can be created, shared, copied, and deleted.
- Profile can be selected, or selected and launched.

## Additional UX Notes
- A “Change Default Home” action is always present in profile manager.
- If no active profile exists, notification shows:
  - “No active profile for {default home}”
  - Tap opens profile manager.
- Mjolnir is described as a **dual-screen home layer** that can be used with or without the system default home.
